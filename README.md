Todo o projeto (site, nome da empresa e dados) são fictícios, tendo apenas o objetivo da criação para o projeto.
Aprendemos a criar layout, design de páginas de site. A Landing Page foi criada do zero, passo-a-passo.
Aprendemos que um site segue toda uma etapa de construção, como (home page “primeira dobra”, informações, conteúdo, time-line, call-to-action, depoimentos, FAQ, rodapé).
Escolhemos criar uma agência de turismo, que é sobre algo que eu gosto e me familiarizei muito, que é viajar. Todos os destinos que eu coloquei no projeto são lugares que eu já viajei realmente e adorei. Foi um gatilho criativo que eu tive assistindo aos tutoriais no Youtube. 
Aprendemos a usar interfaces, que dá pra fazer fluxogramas, interações e animações.
Todo o site foi desenvolvido o protótipo no FIGMA, nós fomos seguindo os tutoriais do Youtube e  fica bem mais compreensível e acessível. Tem muitos plug-ins que é possível usar no Figma, que auxilia muito na parte do design e layout do projeto.
O site foi desenvolvido utilizando o Visual Studio, um ambiente de desenvolvimento integrado (IDE) amplamente utilizado para a criação de aplicações web.
O CSS, responsável pela formatação e estilo visual, garante um design atrativo e funcional. Desenvolvemos um sistema Full Stack integrando duas das principais tecnologias do mercado: Front-end e Back-end, com foco em funcionalidade e clareza na navegação.
Optamos por um site One Page, garantindo uma experiência fluida ao usuário.
No Front-end, utilizamos HTML5, CSS e JavaScript, implementando animações visuais que tornam a interface mais dinâmica e intuitiva.
No Back-end, escolhemos PHP para a lógica de servidor, conectado a um banco de dados relacional MySQL.  Adotamos práticas de segurança como a sanitização de senhas antes de armazená-las no banco, oferecendo uma camada de proteção aos dados dos usuários.
Além disso, aplicamos validações de formulários com JavaScript utilizando estruturas condicionais como If/Else, assegurando que os dados inseridos estejam corretos antes de serem enviados ao servidor.
Embora o foco principal tenha sido a experiência em dispositivos desktop, o layout pode ser expandido futuramente com uma abordagem Mobile first, visando acessibilidade em diferentes plataformas.
